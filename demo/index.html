How It Works

Cross-chain in one click

Circle MSCA + Gateway protocol handles everything under the hood.

1

User initiates transfer

Specify recipient, destination chain, and amount. OmniFlow handles routing.

omni.send()
2

MSCA approves & burns

Smart account approves USDC and calls Circle Gateway to burn tokens on source chain.

depositForBurn()
3

Attestation & mint

Circle attests the burn. Gateway mints equivalent USDC on destination chain.

receiveMessage()
4

Route optimization

LI.FI integration finds the cheapest route when direct Gateway path isn't available.

lifi.getQuote()
5

Batch execution

Multiple transfers are batched into a single transaction via MSCA's modular execution.

executeBatch()
6

Delivery confirmation

Real-time status tracking. Funds arrive on destination chain, ready to use.

status: "completed"

Features

Built for modern payments

Everything you need to move money across chains without friction.

Gasless Transfers

Users never pay gas. Paymaster sponsors all transactions through ERC-4337 infrastructure.

Non-custodial

Your keys, your funds. MSCA smart accounts keep you in full control at all times.

Modular Architecture

Plug-and-play input/output modules. Swap bridges, add new chains, or integrate DEXes.

Batch Payouts

Send to multiple recipients on different chains in a single transaction. Perfect for payroll.

ENS Resolution

Send to human-readable names. Resolve ENS on any chain automatically.

Chain Abstraction

Users don't need to know which chain they're on. OmniFlow handles routing transparently.

Developer API

Ship cross-chain payments in minutes

A clean, typed SDK that handles all the complexity of cross-chain USDC transfers. Batch payouts, ENS resolution, and gas sponsorship — all in a single call.

  • TypeScript-first with full type safety
  • Built-in ENS & address resolution
  • Gasless via ERC-4337 Paymaster
  • Real-time transfer status tracking
